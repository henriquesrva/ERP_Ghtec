# `docs/FEATURES_LOG.md`

```md
# FEATURES_LOG.md

## Função deste arquivo

Este arquivo registra funcionalidades planejadas, em desenvolvimento, pausadas ou concluídas do sistema.

Ele serve para organizar a evolução do projeto e impedir que ideias, decisões, critérios de aceite e pendências se percam durante o desenvolvimento.

Este arquivo é o painel de evolução do sistema.

---

## Instruções para humanos e IA

Use este arquivo para registrar features relevantes.

Não registre aqui bugs pontuais, logs de erro, prompts ou alterações pequenas sem impacto real no produto.

Ao implementar ou alterar uma feature, não edite este arquivo automaticamente sem necessidade.

Ao final de uma tarefa, analise se a alteração muda o status de alguma feature existente ou cria uma nova feature relevante.

Se fizer sentido atualizar este arquivo, sugira o trecho exato a ser adicionado ou alterado.

Use linguagem objetiva, curta e verificável.

---

## O que deve entrar neste arquivo

- Features novas planejadas
- Melhorias relevantes em funcionalidades existentes
- Mudanças relevantes de comportamento do sistema
- Critérios de aceite de cada feature
- Status da implementação
- Pendências conhecidas
- Decisões relacionadas a funcionalidades
- Próximos passos de produto

---

## O que NÃO deve entrar neste arquivo

- Bugs pontuais
- Erros de terminal
- Ajustes pequenos de CSS
- Prompts usados com IA
- Contexto geral permanente do sistema
- Decisões arquiteturais já consolidadas
- Histórico completo de commits
- Dados sensíveis de clientes
- Tokens, senhas ou chaves de API
